## 使用方法

### 1. 命令行使用 curl

```bash
# 使用 SOCKS5 代理
curl -x socks5://127.0.0.1:1080 https://www.example.com

# 下载文件
curl -x socks5://127.0.0.1:1080 -O https://example.com/file.zip
```

### 2. 浏览器配置

**Chrome/Edge：**
1. 设置 → 系统
2. 打开"打开您计算机的代理设置"
3. 手动配置代理：
   - SOCKS 主机：127.0.0.1
   - 端口：1080

**Firefox：**
1. 设置 → 常规 → 网络设置
2. 手动配置代理：
   - SOCKS 主机：127.0.0.1
   - 端口：1080
   - SOCKS v5

### 3. Git 配置（可选）

```bash
# 为 Git 配置 SOCKS5 代理
git config --global http.proxy 'socks5://127.0.0.1:1080'
git config --global https.proxy 'socks5://127.0.0.1:1080'

# 取消 Git 代理
git config --global --unset http.proxy
git config --global --unset https.proxy
```

### 4. SSH 配置（可选）

```bash
# 使用 SSH 隧道通过 Git
ssh -o ProxyCommand="nc -X 5 -x 127.0.0.1:1080 %h %p" user@github.com
```

## 注意事项

### 使用场景

**推荐使用 SSH 隧道的情况：**
- 直连访问国外网站失败
- 访问被限制的资源
- 需要通过 VPS 中转流量

**不推荐使用 SSH 隧道的情况：**
- 访问国内网站（直连更快）
- 访问 GitHub（直连快3倍）
- 一般网络浏览

### 性能对比

| 访问类型 | 直连 | SSH 隧道 |
|---------|------|---------|
| 国内网站 | 快 | 慢（不必要的开销） |
| GitHub HTTPS | 快（0.35s） | 慢（1.09s） |
| 国外受限网站 | ❌ 无法访问 | ✅ 可访问 |
