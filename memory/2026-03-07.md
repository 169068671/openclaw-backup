## 18:02 - VPS Google 访问测试

### 测试结果
| 测试方式 | 结果 | 说明 |
|---------|------|------|
| VPS 直连 Google | ✅ HTTP/2 200 | VPS 可以正常访问 Google |
| 本地直连 Google | ❌ 被墙 | Google 在中国被屏蔽 |
| SSH 隧道代理（socks5://） | ❌ 超时 | DNS 解析问题 |
| SSH 隧道代理（socks5h://） | ✅ HTTP 200 | 远程服务器解析 DNS |

### 问题根源

**DNS 解析不一致**：
- 本地 DNS 解析：www.google.com → 199.16.158.8
- VPS DNS 解析：www.google.com → 172.217.26.132
- Google 使用 GeoDNS（根据地理位置返回不同的 IP）

**两种 SOCKS5 代理模式**：
- `socks5://` - 本地解析 DNS，然后连接远程 IP（可能失败）❌
- `socks5h://` - 远程服务器解析 DNS，返回域名让远程连接（成功）✅

### 解决方案

**修复 ssh-tunnel 技能**：
1. 将 `socks5://` 改为 `socks5h://`
2. 更新 SKILL.md 和 README.md 文档
3. 测试验证 Google 访问成功

**更新命令**：
```bash
# 修复前（失败）
curl --proxy socks5://127.0.0.1:1080 -I https://www.google.com

# 修复后（成功）
curl --proxy socks5h://127.0.0.1:1080 -I https://www.google.com
```

### Git 提交

- 提交 ID：663d9da
- 提交信息：修复 ssh-tunnel 技能：使用 socks5h:// 确保远程服务器解析 DNS
- 文件：skills/ssh-tunnel (软链接)

### 重要提醒

**使用 SOCKS5 代理时的注意事项**：
- 使用 `socks5h://` 确保远程服务器解析 DNS
- 避免使用 `socks5://`（本地解析，可能导致访问失败）
- 对于 Google 等使用 GeoDNS 的服务，这一点尤为重要

### 代理配置示例

```bash
# 终端临时代理（推荐）
export ALL_PROXY=socks5h://127.0.0.1:1080
export HTTP_PROXY=socks5h://127.0.0.1:1080
export HTTPS_PROXY=socks5h://127.0.0.1:1080

# yt-dlp（YouTube 下载）
yt-dlp --proxy socks5h://127.0.0.1:1080 "URL"

# curl
curl --proxy socks5h://127.0.0.1:1080 URL
```
