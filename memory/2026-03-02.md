## 🎯 关键决策

### 1. 使用 SSH 隧道代替 Tinyproxy

**原因：**
- Tinyproxy HTTPS 代理不稳定
- 经常超时和连接重置
- 用户体验差（页面一直 loading）

**决策：**
- 切换到 SSH 隧道（SOCKS5）
- 更稳定、更可靠
- 配置更简单

### 2. OpenClaw 直连国内网络

**原则：**
- OpenClaw 不设置代理
- 所有 AI 模型调用走直连
- 钉钉通道走直连

**结果：**
- OpenClaw 性能不受代理影响
- AI 响应速度快

### 3. 分层架构

**设计：**
- 国内流量 → 直连（高速）
- 国外流量 → 代理（慢速但可用）

**优势：**
- 不影响国内访问速度
- 实现国外网站访问
