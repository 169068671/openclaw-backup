## 🚀 手动启动方法

### 方法 1：使用启动脚本（推荐）

**启动命令：**
```bash
bash ~/openclaw/workspace/start-ssh-tunnel.sh
```

**脚本功能：**
- ✅ 检查是否已有 SSH 隧道在运行
- ✅ 自动停止旧隧道
- ✅ 启动新隧道
- ✅ 显示隧道状态
- ✅ 显示使用方法

**输出示例：**
```
=========================================
SSH 隧道手动启动
=========================================
[1/2] 启动 SSH 隧道...
[2/2] ✅ SSH 隧道已启动（PID: 51699）

隧道信息：
  协议：SOCKS5
  地址：127.0.0.1:1080
  目标：VPS (76.13.219.143)

使用方法：
  浏览器配置：127.0.0.1:1080 (SOCKS v5)
  测试命令：curl --socks5 127.0.0.1:1080 https://www.google.com

=========================================
完成
=========================================
```

### 方法 2：直接命令

**启动命令：**
```bash
sshpass -p 'Whj001.Whj001' ssh -N -D 1080 -f root@76.13.219.143
```

**参数说明：**
- `-N`：不执行远程命令，只做端口转发
- `-D 1080`：创建 SOCKS5 代理，监听本地 1080 端口
- `-f`：后台运行
- `root@76.13.219.143`：目标服务器

## 🛑 停止 SSH 隧道

### 方法 1：使用脚本

脚本会自动检查并停止旧隧道。

### 方法 2：手动停止

```bash
# 查找隧道进程
pgrep -f "ssh.*1080"

# 停止隧道
pkill -f "ssh.*1080"

# 或使用 kill 命令
kill <PID>
```

### 方法 3：停止服务（不推荐）

```bash
# 停止服务（服务已禁用，不会自动启动）
systemctl stop ssh-tunnel.service
```

## 📊 隧道状态检查

### 检查是否在运行

```bash
# 方法 1：查看进程
pgrep -f "ssh.*1080"

# 方法 2：查看端口
netstat -tlnp | grep 1080
# 或
ss -tlnp | grep 1080
```

### 查看隧道信息

```bash
# 查看进程信息
ps aux | grep "ssh.*1080"

# 查看连接
netstat -tlnp | grep 1080
```

## 🎯 使用场景

### 何时启动 SSH 隧道

1. **需要访问国外网站时**
   - 启动隧道
   - 配置浏览器使用代理
   - 访问国外网站

2. **需要下载国外资源时**
   - 启动隧道
   - 使用 curl/wget 下载
   - 或在浏览器中下载

3. **需要克隆 GitHub 仓库时**
   - 启动隧道
   - 使用 git clone
   - 镜像拉取更快

### 何时停止 SSH 隧道

1. **不需要访问国外网站时**
   - 停止隧道（节省资源）
   - 国内网站直连更快

2. **使用本地代理时**
   - 如果使用其他代理工具
   - 停止 SSH 隧道避免冲突

3. **系统资源紧张时**
   - 停止隧道释放资源

## 💡 使用建议

### 推荐使用场景

1. **按需启动**
   - 需要时启动，不需要时停止
   - 节省系统资源
   - 减少网络开销

2. **临时使用**
   - 下载大型文件时启动
   - 访问国外网站时启动
   - 使用完后立即停止

### 性能优化

**启动选项：**
```bash
# 添加压缩（加速）
ssh -N -D 1080 -C -f root@76.13.219.143

# 指定加密算法（更快）
ssh -N -D 1080 -c aes128-gcm -f root@76.13.219.143

# 连接保活
ssh -N -D 1080 -o ServerAliveInterval=60 -o ServerAliveCountMax=3 -f root@76.13.219.143
```
