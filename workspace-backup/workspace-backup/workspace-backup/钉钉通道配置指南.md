## 🎯 钉钉应用配置要求

### 1. 钉钉开放平台设置

**应用类型：** 企业内部开发

**机器人能力配置：**

| 配置项 | 要求 |
|--------|------|
| 消息接收模式 | **Stream 模式**（⚠️ 不是 Webhook） |
| 机器人类型 | 企业内部机器人 |

### 2. 必需权限

请确保你的钉钉应用已开通以下权限：

| 权限名称 | 用途 | 是否必需 |
|---------|------|---------|
| `Card.Streaming.Write` | AI Card 流式响应 | ✅ 必需 |
| `Card.Instance.Write` | AI Card 实例写入 | ✅ 必需 |
| `qyapi_robot_sendmsg` | 机器人发送消息 | ✅ 必需 |

### 3. 发布应用

**重要：** 配置完成后，必须**发布应用**才能生效。

1. 在钉钉开放平台进入应用管理
2. 点击"发布应用"
3. 选择发布版本
4. 等待发布完成

## 🚀 使用指南

### 用户侧使用方式

#### 1. 添加机器人到钉钉

**添加到个人聊天：**
- 在钉钉中搜索你的机器人（通过 AppKey 或应用名称）
- 发起单聊即可开始对话

**添加到群聊：**
- 在群设置中添加机器人
- @机器人即可对话

#### 2. 会话命令

用户可以发送以下命令控制会话：

| 命令 | 效果 |
|------|------|
| `/new` | 开启新会话（清空对话历史） |
| `/reset` | 开启新会话（清空对话历史） |
| `/clear` | 开启新会话（清空对话历史） |
| `新会话` | 开启新会话（清空对话历史） |
| `重新开始` | 开启新会话（清空对话历史） |
| `清空对话` | 开启新会话（清空对话历史） |

#### 3. 功能特性

- ✅ **AI Card 流式响应** - 打字机效果，实时显示 AI 回复
- ✅ **会话持久化** - 同一用户的多轮对话共享上下文
- ✅ **超时自动新会话** - 默认 30 分钟无活动自动开启新对话
- ✅ **图片自动上传** - 本地图片路径自动上传到钉钉
- ✅ **双向通信** - 支持用户发消息和 OpenClaw 主动推送消息

## ⚠️ 常见问题排查

### 问题 1：钉钉机器人无响应

**可能原因：**

1. **应用未发布**
   - 检查：在钉钉开放平台查看应用状态
   - 解决：点击"发布应用"并等待发布完成

2. **消息接收模式错误**
   - 检查：确认是 **Stream 模式**，不是 Webhook
   - 解决：在机器人能力设置中切换为 Stream 模式

3. **权限未开通**
   - 检查：查看应用权限列表
   - 解决：开通 `Card.Streaming.Write`、`Card.Instance.Write`、`qyapi_robot_sendmsg` 权限

4. **AppKey/AppSecret 错误**
   - 检查：核对 `openclaw.json` 中的配置
   - 解决：从钉钉开放平台重新复制正确的 AppKey 和 AppSecret

5. **Gateway 未运行**
   - 检查：`curl http://127.0.0.1:18789/`
   - 解决：`openclaw gateway restart`

### 问题 2：只有纯文本，没有 AI Card

**原因：** 权限未开通或配置错误

**解决方案：**
1. 确认开通权限：
   - `Card.Streaming.Write` ✅
   - `Card.Instance.Write` ✅
2. 重新发布应用
3. 重启 Gateway：`openclaw gateway restart`

### 问题 3：出现 401 错误

**原因：** Gateway 认证失败

**解决方案：**
1. 检查 `gateway.auth.token` 是否与配置文件一致
2. 确认 token 正确复制
3. 重启 Gateway

### 问题 4：出现 405 错误

**原因：** chatCompletions 端点未启用

**解决方案：**
1. 检查 `openclaw.json`：
   ```json
   {
     "gateway": {
       "http": {
         "endpoints": {
           "chatCompletions": {
             "enabled": true  // ✅ 确认为 true
           }
         }
       }
     }
   }
   ```
2. 如果为 false，修改为 true
3. 重启 Gateway

### 问题 5：OpenClaw 无法主动发送消息到钉钉

**可能原因：**
1. 用户 ID 或群 ID 错误
2. 权限不足
3. 网络问题

**解决方案：**
1. 确认正确的用户 ID 或群 ID
2. 检查权限配置
3. 查看日志：`journalctl -u openclaw-gateway -f | grep -i dingtalk`

### 问题 6：会话上下文丢失

**原因：**
1. 会话超时（默认 30 分钟）
2. 用户发送了 `/new` 等会话重置命令
3. Gateway 重启导致内存中的会话丢失

**解决方案：**
1. 调整会话超时时间（配置 `sessionTimeout`）
2. 提醒用户会话会自动重置
3. 考虑使用持久化存储会话
